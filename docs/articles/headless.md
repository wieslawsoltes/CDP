# Headless Test Helpers

When writing in-process headless UI automation tests for Avalonia applications (for example, using `Avalonia.Headless.XUnit` or `Avalonia.Headless`), you frequently need to simulate user interactions such as mouse clicks, dragging-and-dropping, and keyboard inputs on specific UI controls.

The **`Chrome.DevTools.Automation.Headless`** package provides reusable window extension helper methods that eliminate code duplication and simplify headless UI test automation.

---

## 1. Installation

To use these helpers in your test project, add the NuGet package:

```bash
dotnet add package Chrome.DevTools.Automation.Headless --prerelease
```

Make sure your test project also references the core Avalonia test packages:

```xml
<PackageReference Include="Avalonia.Headless.XUnit" Version="12.0.5" />
<PackageReference Include="Avalonia.Headless" Version="12.0.5" />
```

---

## 2. Headless Helper APIs

The package exposes the `CdpHeadlessHelper` static class containing extension methods for the `Avalonia.Controls.Window` class:

### `window.ClickControl(control, button, modifiers)`
Simulates a mouse down and mouse up sequence at the center coordinates of the specified target control relative to the window.
- **`control`**: The target Avalonia `Control` to click.
- **`button`**: The mouse button (e.g. `MouseButton.Left`, `MouseButton.Right`).
- **`modifiers`**: Keyboard modifiers (e.g. `RawInputModifiers.Control`, `RawInputModifiers.Shift`).

### `window.DragAndDrop(source, target)`
Simulates a drag-and-drop mouse sequence starting at the center coordinates of the `source` control, moving to the center coordinates of the `target` control, and releasing the mouse button.
- **`source`**: The control to start dragging.
- **`target`**: The destination control to drop onto.

## 3. Writing Headless Tests

Here is a complete example of a headless test using these helper methods:

```csharp
using System;
using System.Threading.Tasks;
using Avalonia.Controls;
using Avalonia.Headless;
using Avalonia.Headless.XUnit;
using Avalonia.Input;
using CDP.Automation.Headless;
using Xunit;

namespace HeadlessRecordedTests
{
    public class RecordedTests
    {
        [AvaloniaFact]
        public async Task TestButtonClickFlow()
        {
            // 1. Initialize and show the window
            var window = new CdpSampleApp.MainWindow();
            window.Show();

            // 2. Settle layout
            await Task.Delay(100);

            // 3. Find target button
            var button = SelectorEngine.QuerySelector(window, "#btnClickMe") as Control;
            Assert.NotNull(button);

            // 4. Click the button using the headless helper
            window.ClickControl(button, MouseButton.Left, RawInputModifiers.None);

            // 5. Assert result state
            var statusLabel = SelectorEngine.QuerySelector(window, "#lblStatus") as TextBlock;
            Assert.NotNull(statusLabel);
            Assert.Equal("Clicked!", statusLabel.Text);
        }
    }
}
```

## 4. Code Generation Support

The Test Studio recorder automatically uses these helpers when exporting test code for the **Avalonia Headless** format. Rather than generating complex coordinate calculations and private helper methods in every test file, it generates clean, standardized calls to the `Chrome.DevTools.Automation.Headless` package:

```csharp
// Generated by Test Studio:
using CDP.Automation.Headless;

// ...
var element_0 = SelectorEngine.QuerySelector(window, "#btnSubmit") as Control;
Assert.NotNull(element_0);
window.ClickControl(element_0, MouseButton.Left, RawInputModifiers.None);
```
